# 65C816 CPU Reference (SNES 5A22 core)

Sources: snes.nesdev.org wiki (CPU vectors, MVN/MVP), Bruce Clark "65C816 Opcodes" (6502.org, endorsed by nesdev wiki), fullsnes (nocash), 6502.org "Decimal Mode" tutorial. All cycle counts are **CPU cycles**; on the SNES each CPU cycle costs 6, 8 or 12 master clocks depending on the address accessed (see "SNES bus speed" at end). Internal (non-bus) cycles cost 6 master clocks.

## Registers

| Reg | Size | Description |
|---|---|---|
| A (C) | 8/16 | Accumulator. `C` = full 16 bits; `B` = high byte (always accessible via XBA even when m=1) |
| X, Y | 8/16 | Index registers. When x=1 (or E=1) the high bytes XH/YH are **forced to $00** |
| S | 16 | Stack pointer, points to next FREE byte. E=1 forces SH=$01 |
| D | 16 | Direct page base. dp operand `nn` addresses $00:(D+nn) |
| DBR (DB, B) | 8 | Data bank — default bank for abs/indirect data accesses |
| PBR (PB, K) | 8 | Program bank — bank of all instruction fetches |
| PC | 16 | Program counter (PBR:PC = 24-bit fetch address; increments wrap within PBR) |
| P | 8 | Status flags (below) |
| E | 1 | Emulation flag, hidden; only accessible via XCE (swaps C and E) |

## P flags

| Bit | Mask | Native (E=0) | Emulation (E=1) |
|---|---|---|---|
| 7 | $80 | N negative | N |
| 6 | $40 | V overflow | V |
| 5 | $20 | M: 1=8-bit A/memory, 0=16-bit | unused, reads 1 |
| 4 | $10 | X: 1=8-bit X/Y, 0=16-bit (XH/YH forced $00 when 1) | B break: pushed as 1 by BRK/PHP, 0 by IRQ/NMI |
| 3 | $08 | D decimal mode (ADC/SBC BCD) | D |
| 2 | $04 | I IRQ disable | I |
| 1 | $02 | Z zero | Z |
| 0 | $01 | C carry (SBC/CMP: set = no borrow / above-or-equal) | C |

- E=1 forces M=1, X=1 (bits 5/4 become B/unused), SH=$01, XH=YH=$00.
- PLP/RTI with x becoming 1 clears XH/YH. REP/SEP cannot touch E.
- B flag exists only in the pushed copy of P; distinguish BRK from IRQ via the stacked value.

## Reset state

| Item | Value after /RESET |
|---|---|
| E | 1 (always resets into emulation mode) |
| M, X | 1 (forced by E=1) |
| D flag | 0 (cleared) ; I = 1 ; N/V/Z/C undefined |
| D register | $0000 |
| DBR, PBR | $00, $00 |
| SH | $01 (SL not initialized by hardware; emulators commonly use S=$01FF) |
| XH, YH | $00 |
| PC | word at [$00FFFC] |

## Addressing modes — effective addresses and wrap rules

Notation: `nn`=1-byte operand, `nnnn`=2-byte, `nnnnnn`=3-byte; `ptr16/ptr24`=value read at pointer address; DBR=`rr`; K=PBR. "24-bit add" = carries into the bank byte (crosses banks); "16-bit, bank 0" = wraps at $FFFF within bank $00.

| Mode | Bytes | Pointer fetched at | Data address | Wrap |
|---|---|---|---|---|
| Immediate | 2/3 | — | K:PC+1 (+2) | operand fetch wraps in bank K |
| Absolute `nnnn` | 3 | — | DBR:nnnn (+1 for high byte, 24-bit) | data may cross into DBR+1 |
| Absolute,X / ,Y | 3 | — | (DBR<<16 \| nnnn) + X/Y, 24-bit add | crosses banks, never wraps 64K |
| Absolute long `nnnnnn` | 4 | — | nnnnnn (24-bit) | +1 crosses banks |
| Long,X | 4 | — | nnnnnn + X, 24-bit add | crosses banks |
| Direct `nn` | 2 | — | $00:(D+nn), 16-bit, bank 0 | E=1 & DL=$00 & old opcode: $00:DH:((nn)&$FF) page-wraps |
| Direct,X / ,Y | 2 | — | $00:(D+nn+X), 16-bit, bank 0 | E=1 & DL=$00: low byte wraps at 8 bits ($00:DH:(nn+X)&$FF) |
| (dir) | 2 | $00:(D+nn), 16-bit bank 0 | DBR:ptr16, 24-bit for high byte | ptr wraps bank 0; E=1&DL=0: ptr hi at DH:((nn+1)&$FF) |
| (dir,X) | 2 | $00:(D+nn+X), 16-bit bank 0 | DBR:ptr16 | same emu page-wrap rule with +X |
| (dir),Y | 2 | $00:(D+nn), 16-bit bank 0 | (DBR<<16 \| ptr16) + Y, 24-bit add | ptr wraps bank 0 / emu page; data crosses banks |
| [dir] | 2 | $00:(D+nn) .. +2, 16-bit bank 0 | ptr24 | never page-wraps (new mode); ptr wraps bank 0 |
| [dir],Y | 2 | $00:(D+nn) .. +2, bank 0 | ptr24 + Y, 24-bit add | data crosses banks |
| Stack relative `nn,S` | 2 | — | $00:(S+nn), 16-bit, bank 0 | never page-wraps (new mode) |
| (nn,S),Y | 2 | $00:(S+nn), 16-bit bank 0 | (DBR<<16 \| ptr16) + Y, 24-bit add | data crosses banks |
| (abs) — JMP only | 3 | $00:nnnn, +1 wraps bank 0 | K:ptr16 | no page-wrap bug (unlike NMOS 6502) |
| [abs] — JML only | 3 | $00:nnnn .. +2, wraps bank 0 | ptr24 (sets PBR) | |
| (abs,X) — JMP/JSR | 3 | K:(nnnn+X), 16-bit, wraps in bank K | K:ptr16 | |
| rel8 | 2 | — | K:(PC+2+disp8), 16-bit | signed; wraps in bank K; +1 cyc taken; +1 if E=1 and page crossed |
| rel16 (BRL/PER) | 3 | — | K:(PC+3+disp16), 16-bit | wraps in bank K |
| src,dest (MVN/MVP) | 3 | — | src = SS:X, dest = TT:Y | X and Y each wrap within their bank |
| Accumulator / Implied | 1 | — | — | |

Wrap summary: **confined to bank 0** = direct page, stack, (abs)/[abs] pointers. **Confined to bank K** = (abs,X) pointer, PC/branches. **MVN/MVP** wrap in src and dest banks. Everything else (data of abs,X, long,X, (dir),Y, [dir],Y, 16-bit data straddling $xxFFFF, …) uses full 24-bit arithmetic and crosses banks. Page wrapping exists **only** in emulation mode for "old" (65C02-era) instructions: direct page when DL=$00, and the stack in page 1.

## Cycle-formula legend (used in the opcode table)

| Var | Meaning |
|---|---|
| m | value of M flag (1 or 0) — e.g. `4-m` = 4 cycles if m=1, 5 if m=0 |
| x | value of X flag |
| w | 1 if DL (low byte of D) != 0, else 0 |
| p | 1 if index addition crosses a page boundary, else 0 (applies to read ops with `x*p` term; stores/RMW always pay the cycle) |
| t | 1 if branch taken |
| e | value of E flag (1=emulation) |

Common patterns: RMW dir = `7-2m+w` (5 base, +2 if 16-bit, +1 if DL≠0); RMW abs,X = `9-2m` (always 7 at m=1 — no page-cross discount); read abs,X = `6-m-x+x*p` (4 at m=x=1 no cross, +1 on cross or if x=0); store abs,X = `6-m` (always). The 65C816 performs an internal cycle instead of the NMOS 6502 dummy write in RMW ops, and decimal mode adds **no** extra cycle.

## Full 256-opcode table

Len `3-m`/`3-x` = 3 bytes when the flag is 0 (16-bit immediate), 2 when 1. Flags column = `nvmxdizc` positions; `m`/`x` mark flag-width-dependent result flags, `0`/`1` = forced value, `.` = unaffected, `*` = affected (value not tied to a flag-width bit position; for PLP/RTI/REP/SEP: all 8 flags writable). BRK/COP are assembled as 1/2 bytes but always push PC+2 (signature byte skipped on return). MVN/MVP: 7 cycles **per byte moved**. Branches: `2+t+t*e*p`.

## Interrupt sequences

| Event | Cycles | Sequence |
|---|---|---|
| NMI / IRQ / ABORT | 8-e (7 emu, 8 native) | 2 internal cycles; **native**: push PBR, push PCH, push PCL, push P; **emu**: push PCH, PCL, P (B=0 in pushed P); then I=1, D=0, PBR=$00, PC=vector |
| BRK ($00) / COP ($02) | 8-e | fetch opcode, fetch signature byte (PC+2 is pushed); then as above (emu BRK pushes P with B=1) |
| RESET | — | E=1, see reset state; PC=[$00FFFC] |

- I is set and **D is cleared** after pushing P (all interrupts, incl. COP/ABORT).
- IRQ is level-sensitive: taken between instructions whenever /IRQ low and I=0. NMI is edge-sensitive (high→low). Current instruction always completes first (except ABORT/RESET) — the sole exception is MVN/MVP (interruptible per byte, below).
- Emulation mode: interrupts push only 16-bit PC (no PBR) and RTI pulls 3 bytes; the stack pushes wrap in page 1 (COP and ABORT behave as "old" here too). Handler must RTI in the same mode the interrupt was taken in.
- Stacked-P example (native, S=$01FF, NMI during code at $345678): $34→$0001FF, PCH, PCL, P → S=$01FB.

## RTI / RTS / RTL / JSR / JSL / JMP / JML semantics

| Op | Behavior |
|---|---|
| JMP $4C | PC=nnnn, PBR unchanged |
| JMP $5C (JML) | PBR:PC = nnnnnn |
| JMP ($6C) | PC = word[$00:nnnn] (pointer wraps bank 0, no page bug), PBR unchanged |
| JMP ($7C) (abs,X) | PC = word[PBR:(nnnn+X)] |
| JMP [$DC] (JML) | PBR:PC = 24-bit ptr at $00:nnnn |
| JSR $20 | push16 (addr of JSR + 2) hi,lo — i.e. next instruction minus 1; PC=nnnn |
| JSL $22 | push8 PBR, push16 (addr of JSL + 3); PBR:PC=nnnnnn. "New" op: no page-1 stack wrap in emu |
| JSR ($FC) | push16 (addr + 2); PC = word[PBR:(nnnn+X)] |
| RTS $60 | PC = pull16 + 1, PBR unchanged |
| RTL $6B | PC = pull16 + 1, then PBR = pull8 |
| RTI $40 | P = pull8, PC = pull16, then PBR = pull8 iff E=0. PC **not** incremented |

## Emulation mode (E=1) differences

1. M=X forced 1 (8-bit A/X/Y); bits 5:4 of P are unused/B; XH=YH=$00; SH=$01.
2. Stack wraps within page 1 ($0100-$01FF) for "old" instructions and **all interrupts** (incl. COP/ABORT). "New" instructions (PEA, PEI, PER, PHD, PLD, PHB→pull PLB, JSL/RTL, stack-relative modes, JSR (abs,X)) do **not** wrap: S can temporarily leave page 1 mid-instruction, but SH is forced back to $01 at the end. (E.g. S=$0100: PHP pushes to $000100 → S=$01FF; PEA pushes hi to $000100, lo to $0000FF → S=$01FE.)
3. Direct page: if DL=$00, old opcodes wrap dp+index and dp pointer fetches within page $00:DH00-DHFF (LDA $FF,X with X=1, D=0 reads $000000; LDA ($FF) reads ptr hi from $DH00). If DL≠$00, no wrap (16-bit D+nn, bank 0). New modes ([dp], PEI, etc.) never page-wrap.
4. BRK/IRQ share vector $FFFE; distinguish via stacked B (bit 4): 1=BRK/PHP, 0=IRQ/NMI.
5. Taken branches crossing a page: +1 cycle only when E=1.
6. TXS sets SL only (SH stays $01); in native x=1 mode TXS sets S=$00XX instead (XH=0) — a real 8-bit/native difference.
7. abs,X and (dp),Y data addresses still cross banks (no 64K wrap) — differs from a real 6502.
8. Interrupts/RTI: 16-bit PC only (no PBR byte).

## Decimal mode ADC/SBC (exact 65C816 algorithm, per 6502.org Decimal Mode tutorial)

No extra cycle. 8-bit ADC (`A = A+B+C`), unlimited-precision unsigned binary arithmetic:

```
AL = (A & $0F) + (B & $0F) + C
if AL >= $0A: AL = ((AL + $06) & $0F) + $10
A' = (A & $F0) + (B & $F0) + AL          ; may exceed $FF
V  = computed HERE, on the signed (two's-complement) value of that sum,
     BEFORE the +$60 correction: set iff sum < -128 or > 127
if A' >= $A0: A' = A' + $60
result = A' & $FF ; C = (A' >= $100) ; N = bit7(result) ; Z = (result == 0)
```

8-bit SBC (`A = A-B+C-1`) — 65C816 uses the NMOS-6502 sequence, flags from binary except N/Z:

```
AL = (A & $0F) - (B & $0F) + C - 1
if AL < 0: AL = ((AL - $06) & $0F) - $10
A' = (A & $F0) - (B & $F0) + AL
if A' < 0: A' = A' - $60
result = A' & $FF
C, V = exactly as a BINARY SBC of the same operands (C=1 iff no borrow)
N = bit7(result) ; Z = (result == 0)
```

16-bit (m=0): identical to chaining two 8-bit decimal ops low-byte-then-high-byte with carry between them (equivalently: per-nibble across 4 nibbles, thresholds $A/$A0/$A00/$A000, corrections $06/$60/$600/$6000). N = bit 15 of the final result; Z = whole 16-bit result == 0; C from the top byte; V from the high-byte stage (bit 15, before its $60 correction for ADC / binary V for SBC). Valid BCD range: ADC carry means result > 9999.

CMP/CPX/CPY always use binary subtraction (even when D=1), don't include carry, don't touch V.

## WAI / STP

- **WAI** ($CB, 3 cycles): halts until NMI, IRQ, ABORT or RESET. If an IRQ arrives while I=1, execution simply **continues at the next instruction** (no vector, no push). If I=0, the interrupt is taken immediately (handler sees return address = instruction after WAI).
- **STP** ($DB, 3 cycles): stops the clock; only RESET recovers.

## MVN ($54) / MVP ($44)

- Machine code: `$54/$44, destBank, srcBank` (dest operand byte FIRST; assembler syntax `MVN #src,#dest`).
- A (always full 16-bit, regardless of M/E) = byte count minus 1. X=source offset, Y=dest offset (8-bit if x=1 → confined to page 0 of each bank).
- Per iteration (7 cycles): read [srcBank:X] → write [destBank:Y]; DBR = destBank; MVN: X++, Y++; MVP: X--, Y--; A--; if A != $FFFF, PC = address of the MVx opcode (repeats); else continue. X/Y wrap within their banks.
- MVN: X/Y = lowest address of each block (ascending copy). MVP: X/Y = highest address (descending copy). Use MVP when dest > src overlapping, MVN when dest < src. Overlap abuse = pattern fill.
- Interruptible between bytes (only instruction that is): IRQ/NMI can be serviced every 7th cycle; since PC still points at the MVx instruction, RTI resumes the move. Flags unaffected.
